### 1. 应用界面结构说明
本实验基于 Jetpack Compose 声明式 UI 框架，开发了 Dice Roller 掷骰子交互应用。应用整体采用 Column 布局作为根容器，设置全屏铺满及垂直、水平居中对齐，确保骰子图片与操作按钮始终显示在屏幕正中央，提升用户交互体验。界面核心由三部分组成：一是用于展示当前骰子点数的 Image 组件，合理设置无障碍描述信息；二是用于分隔图片与按钮的 Spacer 组件，优化界面视觉间距；三是用于触发掷骰子操作的 Button 组件，按钮文本设置为“Roll”，交互逻辑清晰直观。所有界面元素均封装在独立的可组合函数中，结构模块化，便于维护与扩展。

### 2. 如何使用 Compose 状态保存骰子结果
为实现骰子点数的持久化保存与界面自动刷新，本应用采用 Jetpack Compose 提供的状态管理机制。在可组合函数中，通过 `remember` 配合 `mutableStateOf` 声明可观察状态变量 `result`，初始值设为 1，用于存储当前骰子点数。其中，`remember` 用于在界面重组时缓存状态，避免数据丢失；`mutableStateOf` 用于创建可变状态，当状态值发生变化时，会自动触发依赖该状态的 UI 组件重组。用户点击“Roll”按钮时，程序生成 1~6 范围内的随机整数并更新 `result` 状态，Compose 框架自动驱动界面刷新，实现状态与 UI 的同步更新，完全符合声明式 UI 的设计理念。

### 3. 如何根据点数切换图片资源
为实现骰子点数与图片资源的一一对应，提前在项目 `res/drawable` 目录中导入 6 张对应点数 1~6 的骰子图片资源，命名规范为 `dice_1` 至 `dice_6`。在代码中，通过 `when` 条件表达式对状态变量 `result` 的值进行判断，为每个点数匹配唯一对应的图片资源 ID，同时设置 `else` 兜底分支，确保逻辑完整性，避免出现资源匹配异常。当 `result` 状态更新时，图片资源 ID 同步变化，Image 组件自动加载并渲染对应的骰子图片，实现点数与图片的无缝切换，无需手动操作视图刷新。

### 4. 设置了哪些断点，分别观察了什么
本次实验严格按照实验要求，在核心逻辑位置设置了 2 个断点，用于观察程序执行流程与变量变化：
1.  第一个断点设置在 `val imageResource = when(result)` 代码行，核心观察目标为：当前状态变量 `result` 的具体取值，验证随机数生成的有效性；同时观察 `imageResource` 的赋值结果，确认点数与图片资源的映射逻辑是否正确，确保不同点数能匹配到对应的骰子图片。
2.  第二个断点设置在 Button 组件的 `onClick` 点击事件内部，即 `result = (1..6).random()` 赋值语句处，核心观察目标为：按钮点击事件是否正常触发，随机数是否能在 1~6 范围内正常生成，以及状态变量 `result` 是否能被成功更新，跟踪状态变化的完整流程。

### 5. Step Into、Step Over、Step Out 的使用体会
在本次调试过程中，熟练运用了 Android Studio 调试器的三个核心操作，具体体会如下：
- Step Into（进入函数）：当程序停留在断点处时，使用该操作可进入当前执行的可组合函数内部，能够细致跟踪界面初始化、状态读取与资源映射的完整流程，直观理解 Compose 组合函数的执行顺序，便于深入排查函数内部的逻辑细节。
- Step Over（逐行执行）：该操作用于逐行执行当前函数内的代码，不进入子函数，适合在断点处观察每一步代码的执行过程，重点查看 `when` 表达式的分支选择、变量赋值等关键操作，实时跟踪变量值的变化，快速定位潜在逻辑问题。
- Step Out（跳出函数）：当深入函数内部查看完逻辑细节后，使用该操作可快速跳出当前函数，直接返回上一层调用位置，避免逐行执行冗余代码，有效提升调试效率，确保调试流程的连贯性。

### 6. 遇到的问题与解决过程
在应用开发与调试过程中，遇到了部分常见问题，通过逐步排查与优化，均得到有效解决，具体过程如下：
1.  界面内容无法居中显示：初始开发时，根布局使用了自带内边距的容器，导致骰子图片与按钮偏上，无法实现全屏居中。排查后，将根布局替换为全屏 Column 容器，并设置垂直、水平居中对齐，问题得以解决，界面显示符合预期。
2.  调试断点无法命中：启动项目后，设置的断点始终无法触发，程序直接正常运行。经检查发现，误使用普通 Run 模式启动项目，而非 Debug 模式。切换为 Debug 'app' 模式重新启动后，断点正常生效，调试流程顺利推进。
3.  骰子图片无法正常加载：应用运行后，骰子图片显示空白，未正常渲染。排查发现，图片资源文件名与代码中引用的名称不一致，导致资源加载失败。修正图片资源文件名，确保与代码引用完全统一后，图片正常显示，功能恢复正常。
